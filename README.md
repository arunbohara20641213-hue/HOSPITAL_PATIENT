# Hospital Patient Management System
A C-based file handling project for storing, retrieving, and analyzing hospital patient records. The system is split across multiple programs, each handling a different query on the same data file.

---
## Project Structure
```
HOSPITAL_PATIENT/
├── hosp_write.c           # Collects patient data and saves to hospital.txt
├── hosp_read.c            # Reads all records + filters by city
├── hosp_age_analysis.c    # Finds oldest and youngest patient
├── hosp_critical.c        # Filters patients with critical conditions
├── hosp_ward_count.c      # Counts patients per ward
├── output/                # Compiled binaries (generated after build)
└── hospital.txt           # Shared data file (generated by hosp_write)
```

---
## Patient Record Fields
Each record stores the following:

| Field | Description |
|---|---|
| Name | Patient's full name (no spaces) |
| Age | Patient's age |
| Address | City of residence (no spaces) |
| Condition | Medical condition (no spaces) |
| Ward | Ward they are assigned to |

## Requirements
- GCC or any C99-compatible compiler
- Terminal or Command Prompt
- Compatible with Windows, Linux, and macOS

---
## How to Compile

### Compile All Programs
Run these commands from inside the project folder:

```bash
# Individual compilation
gcc hosp_write.c -o hosp_write
gcc hosp_read.c -o hosp_read
gcc hosp_age_analysis.c -o hosp_age_analysis
gcc hosp_critical.c -o hosp_critical
gcc hosp_ward_count.c -o hosp_ward_count
```

Or use a Makefile (if available) for convenience:
```bash
make all
```

---
## How to Run

### Step 1 — Write Patient Data
**Always run this first.** It will prompt you to enter details for 24 patients and saves them to `hospital.txt`.

```bash
./hosp_write        # Linux / macOS
hosp_write.exe      # Windows
```

You will be prompted like this for each patient:
```
Patient 1:
Name: RamBahadur
Age: 45
Address (no spaces): Kathmandu
Condition (no spaces): Fever
Ward: General
```

### Step 2 — Read and Filter by City
Prints all 24 patients, then asks for a city name and shows only matching records:

```bash
./hosp_read         # Linux / macOS
hosp_read.exe       # Windows
```

Expected output:
```
All Patients:
Name          Age    Address      Condition    Ward
RamBahadur    45     Kathmandu    Fever        General
...

Enter city to filter: Kathmandu
Patients from Kathmandu:
Name          Age    Address      Condition    Ward
RamBahadur    45     Kathmandu    Fever        General
Total patients from Kathmandu: 1
```

### Step 3 — Find Oldest and Youngest Patients
Analyzes all records and displays the patient with the highest and lowest age values:

```bash
./hosp_age_analysis    # Linux / macOS
hosp_age_analysis.exe  # Windows
```

Expected output:
```
Age Analysis Report
==================
Oldest Patient:
  Name: PatientName
  Age: 78
  Address: CityName
  Condition: ChestPain
  Ward: ICU

Youngest Patient:
  Name: PatientName
  Age: 18
  Address: CityName
  Condition: Fever
  Ward: General
```

### Step 4 — Filter by Critical Condition
Searches all records for patients with critical medical conditions and lists them separately:

```bash
./hosp_critical    # Linux / macOS
hosp_critical.exe  # Windows
```

This program will prompt for a critical condition keyword and display matching patients:
```
Enter critical condition to search: Severe
Critical Patients:
Name          Age    Address      Condition       Ward
PatientA      56     Kathmandu    SevereFever    ICU
PatientB      72     Pokhara      SevereAsthma   CCU
Total critical patients found: 2
```

### Step 5 — Count Patients Per Ward
Groups all patients by ward and prints a count for each one:

```bash
./hosp_ward_count     # Linux / macOS
hosp_ward_count.exe   # Windows
```

Expected output:
```
Ward-Wise Patient Distribution
=============================
Ward Name          Patient Count
General            8
ICU                5
CCU                4
Pediatric          3
Orthopedic         2
Cardiology         2
Total Patients: 24
```

## Feature Summary

| Feature | Program | Purpose |
|---------|---------|---------|
| **Data Entry** | `hosp_write` | Collects and stores patient records |
| **View All & Filter by City** | `hosp_read` | Lists all patients or filters by city |

---
## Important Notes

- **Execution Order**: Always run `hosp_write` first to generate `hospital.txt`. Other programs depend on this file.
- **Same Directory**: All programs must be run from the same directory as `hospital.txt`.
- **Case Sensitivity**: Filters are case-sensitive. `kathmandu` will not match `Kathmandu`.
- **No Spaces**: Multi-word inputs are not supported. Use formats like:
  - `ChestPain` instead of `Chest Pain`
  - `NewBaneshwor` instead of `New Baneshwor`
- **Fixed Patient Count**: Currently set to 24 patients per session.

## Troubleshooting

### Error: "hospital.txt not found"
**Solution**: Run `hosp_write` first to create the data file.

### Error: "Permission denied"
**Solution**: On Linux/macOS, ensure the executable has permissions:
```bash
chmod +x hosp_write hosp_read hosp_age_analysis hosp_critical hosp_ward_count
```

### No matching records when filtering
- Check spelling and case sensitivity
- Ensure patient data was entered correctly in `hosp_write`
